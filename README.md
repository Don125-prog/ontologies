# ontologies
Этот репозиторий содержит учебный проект для дисциплины "Современные методы решения инженерных задач". Он демонстрирует навыки моделирования онтологий с использованием OWL, Protégé, Apache Jena Fuseki и Postman. 

# task1: Базовая онтология
Создана онтология с минимум 3 классами, 2 типами отношений (объектными и типизированными данными) и 5+ сущностями (индивидами).
Модель проверена в Protégé на consistency и сохранена в RDF/XML.
Пример: иерархия классов с свойствами для демонстрации базовых связей.


# task2
Разработана онтология "Человек" с множеством индивидов X, где каждое X связано свойством "ИмеетРодителя" с уникальным другим X (без самосвязей).
OWL-аксиомы обеспечивают вывод предков и потомков через транзитивные свойства (transitiveObjectProperty) и инверсные связи.
Протестировано в резонере Protégé: DL-запросы возвращают полную родословную.

# task3
Разработан машиночитаемый фрагмент онтологии, включающий логическое правило в виде импликации, заданное на языке SWRL. Это правило служит для автоматической классификации объектов с использованием логического резонера. Классификация решает значимую задачу в выбранной предметной области.

Этапы выполнения:
- Определение предметной области и конкретной задачи классификации, представляющей интерес для данной области.
- Создание онтологии, описывающей выбранную предметную область в контексте задачи классификации, с реализацией в формате Turtle средствами Protégé. Результат сохраняется в первом TTL-файле.
- Формулирование логического правила-импликации, позволяющего классифицировать объекты согласно поставленной задаче, и интеграция этого правила в онтологию с использованием SWRL в Protégé. Итоговая онтология с правилом сохраняется во втором TTL-файле.
- Проверка корректности работы правила с помощью логического резонера (Reasoner) в Protégé.
# Установка и запуск
Protégé
- Скачайте и установите Protégé с официального сайта: protege.stanford.edu
- Откройте файлы онтологий .ttl через меню File → Open (Ctrl+O).
- Запустите резонер: Reasoner → Start reasoner (рекомендуется HermiT или Pellet)
Apache Jena Fuseki
Для установки и запуска сервера Apache Jena Fuseki с поддержкой онтологий выполните следующие шаги:
- Убедитесь, что на вашей системе установлен Docker (и опционально Docker Compose).
- Клонируйте репозиторий с настройками и перейдите в папку проекта:
```sh
git clone https://gitlab.com/study3451345/docker/jena-fuseki-reasoner.git
cd jena-fuseki-reasoner
```
- Разрешите запись на папку проекта для пользователя, от имени которого запускается сервер в контейнере:
```sh
sudo chown -R 100 fuseki
```
- Запустите контейнер одним из способов:
```sh
docker-compose up -d
```
или
```sh
docker run -d --name jena-fuseki-owl2 -p 3030:3030 -v ./fuseki:/fuseki stain/jena-fuseki:4.8.0
```
После запуска сервер будет доступен для загрузки онтологии и выполнению SPARQL-запросы.
Тестирование SPARQL
- Используйте Postman или браузер для запросов к Apache Jena Fuseki
- Пример запроса:
```sh
SELECT * WHERE { ?s ?p ?o } LIMIT 10
```
